# Commands Layer Context

This folder is the Office function-file entrypoint for ribbon commands.

## Files

- `commands.ts`: thin wrappers around shared actions.
- `commands.html`: generated by webpack as the function-file host page for the bundled command script.

## How It Works

- Each ribbon command receives `Office.AddinCommands.Event`.
- The shared helper `withCommandEvent(...)` runs the action, logs the result, and always calls `event.completed()`.
- Command wrappers delegate into `shapeTools.ts`.
- Functions are attached to `window` so Office can invoke them by manifest name.

## Important Binding Rule

`manifest.xml` `<FunctionName>` values must match the names exposed on `window` in `commands.ts`.

If you add or rename a ribbon command:

1. add/update the wrapper
2. expose the wrapper on `window`
3. update the manifest binding

Any mismatch fails at runtime even if the build succeeds.
